# 12 — Internationalization (ES/EN) · Internacionalización (ES/EN)

Goal / Objetivo: a language toggle that switches the whole UI between Spanish and English, and remembers
the choice.

---

## The design

- **EN:** A tiny, dependency-free i18n: a **dictionary** of strings per language, a **React Context**
  that holds the current language and a `t()` function, and a **toggle button** in the navbar. The
  choice is saved in `localStorage`, so it survives refreshes.
- **ES:** Un i18n mínimo y sin dependencias: un **diccionario** de textos por idioma, un **Context de
  React** con el idioma actual y una función `t()`, y un **botón** en la navbar. La elección se guarda en
  `localStorage`, así sobrevive a las recargas.

## The pieces

| File | Role |
|---|---|
| `src/i18n/translations.js` | `{ en: {...}, es: {...} }` — same keys, different values |
| `src/i18n/LanguageContext.jsx` | Provider + `useTranslation()` → `{ lang, setLang, t }` |
| `src/main.jsx` | Wraps `<App/>` in `<LanguageProvider>` |
| `src/components/Navbar.jsx` | The `EN`/`ES` toggle button |

```jsx
const { t, lang, setLang } = useTranslation()
// use it:
<h1>{t('mov.title')}</h1>
<button onClick={() => setLang(lang === 'es' ? 'en' : 'es')}>{lang === 'es' ? 'EN' : 'ES'}</button>
```

## Placeholders

- **EN:** `t()` supports `{param}` placeholders, so dynamic strings translate cleanly:
  `t('irpf.alert', { q: 2, date: '2026-07-20', days: 18 })` →
  *"⏰ Modelo 130 Q2 due on 2026-07-20 — 18 days left"* (or the Spanish equivalent with "T2").
- **ES:** `t()` admite marcadores `{param}`, así que las cadenas dinámicas se traducen sin romperse.

## What's translated

Navbar, page titles/subtitles, buttons, table headers, the chart captions and legend, the VAT labels,
and the IRPF alert/notes — including the quarter prefix (**Q**1 in English, **T**1 in Spanish).

- **EN:** Because the redesign centralized text, wiring i18n was mostly swapping literals for `t('key')`.
  Default language is Spanish; the button flips to English instantly, no reload.
- **ES:** Como el rediseño centralizó el texto, conectar el i18n fue básicamente cambiar literales por
  `t('clave')`. El idioma por defecto es español; el botón cambia a inglés al instante, sin recargar.

## Why it matters for the portfolio

- **EN:** A bilingual UI signals product thinking for international users — a strong fit for remote/EU
  roles, and a natural showcase of your ES/EN profile.
- **ES:** Una interfaz bilingüe demuestra pensamiento de producto para usuarios internacionales — encaja
  con puestos remotos/UE y muestra de forma natural tu perfil ES/EN.

## Verify

```powershell
npm run build   # compiles
```

Open `http://localhost:5173` and click the **EN/ES** button in the navbar — the whole interface switches
language and stays that way after a refresh.
