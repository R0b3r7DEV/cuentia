# 14 — Continuous Integration (GitHub Actions) · Integración continua

Goal / Objetivo: run the backend tests and the frontend build automatically on every push.

> The green "CI passing" badge tells a recruiter, at a glance, that the project actually works and is
> maintained. / La insignia verde de "CI passing" le dice a un reclutador, de un vistazo, que el proyecto
> funciona de verdad y se mantiene.

---

## The workflow

File: `.github/workflows/ci.yml` — two independent jobs:

- **EN — Backend:** checkout → set up PHP 8.4 (with `intl`, `mbstring`, `pdo_pgsql`) → `composer install`
  → `php bin/phpunit`. No database service is needed because the tests are pure units (stubbed
  repository).
- **ES — Backend:** checkout → PHP 8.4 (con `intl`, `mbstring`, `pdo_pgsql`) → `composer install` →
  `php bin/phpunit`. No hace falta base de datos porque los tests son unitarios puros (repositorio con stub).

- **EN — Frontend:** checkout → set up Node 24 (npm cache) → `npm ci` → `npm run build`. This catches any
  broken import or JSX before it reaches the repo.
- **ES — Frontend:** checkout → Node 24 (caché npm) → `npm ci` → `npm run build`. Esto detecta cualquier
  import roto o JSX inválido antes de que llegue al repo.

```yaml
on:
  push: { branches: [master, main] }
  pull_request: { branches: [master, main] }
```

## Why two jobs

- **EN:** The backend and frontend are independent, so they run **in parallel** and fail independently —
  you see exactly which side broke. Each job sets `working-directory` to its folder (monorepo-friendly).
- **ES:** Backend y frontend son independientes, así que corren **en paralelo** y fallan por separado —
  ves exactamente qué lado se rompió. Cada job fija `working-directory` a su carpeta (apto para monorepo).

## The badge

```md
[![CI](https://github.com/R0b3r7DEV/cuentia/actions/workflows/ci.yml/badge.svg)](…/actions/workflows/ci.yml)
```

- **EN:** Added to the README. It turns green when the latest push passes both jobs, red if anything breaks.
- **ES:** Añadida al README. Se pone verde cuando el último push pasa ambos jobs, roja si algo falla.

## Verify

- **EN:** After pushing, open the repo's **Actions** tab: you'll see the `CI` run with the two jobs. Green
  = tests pass and the frontend builds.
- **ES:** Tras hacer push, abre la pestaña **Actions** del repo: verás la ejecución `CI` con los dos jobs.
  Verde = los tests pasan y el frontend compila.
